
*高昂时知行合一，消沉时行知合一。*

*很多时候，过度的抽象并不会让人眼前一亮，可能会让人摸不着头脑。*

*首先考虑使用封装、继承和模板，而不是抽象和重载。*

*可维护、可扩展与简单宜用往往不可兼得，要接受这种矛盾的存在。*
